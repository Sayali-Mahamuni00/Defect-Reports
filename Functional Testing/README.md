This folder contains sample defect reports identified during functional testing
of web-based applications.

## Scope of Testing
- Validation of business logic and application workflows
- Input field validations (positive and negative scenarios)
- UI behavior during user actions
- Form submission and error handling

## What these defect reports demonstrate
- Understanding of functional requirements
- Ability to identify real-world user-impacting issues
- Proper defect logging with clear title, description, and steps
- Correct severity and priority classification
- Expected vs Actual result analysis
- Use of screenshots as supporting evidence

## Example Scenarios Covered
- Invalid input handling
- Missing or incorrect validations
- Workflow-level functional issues

These defect reports were created as part of hands-on manual testing practice
to simulate real-time QA defect reporting standards.
